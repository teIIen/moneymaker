# TODO & Архитектурные планы по развитию Moneymaker2

## 1. Улучшение Strategy Engine (Математика и Индикаторы)
- [ ] **SmokeFX Kraya (Края):** Реализовать индикатор для определения перекупленности/перепроданности и границ каналов.
- [ ] **SmokeFX Tumba:** Добавить логику паттернов тумбы для подтверждения разворотов.
- [ ] **SmokeFX FiboChannel:** Реализовать динамические каналы Фибоначчи для определения целей по тейк-профитам и глобального тренда.
- [ ] **Мульти-таймфреймовый анализ (MTF):** Научить движок смотреть старший таймфрейм (например, 4h) для определения глобального тренда, а сделки открывать по младшему (например, 15m).
- [ ] **Агрегация тиковых данных:** Переход от OHLCV к анализу тиков/кластеров для более точного VSA (Delta, Footprint), если позволит API биржи.

## 2. Risk Management (Риск-менеджмент)
- [ ] **Модуль Position Sizing:** Автоматический расчет размера позиции в зависимости от размера стоп-лосса и заданного риска на сделку (например, 1% от депозита).
- [ ] **Trailing Stop & Breakeven:** Логика подтягивания стоп-лосса вслед за ценой (виртуальный трейлинг).
- [ ] **Дневные лимиты (Daily Drawdown):** Блокировка торговли при достижении лимита убытков за день.

## 3. Execution Core (Исполнение ордеров)
- [ ] **Paper Trading Engine:** Полноценный симулятор торгов с учетом комиссий и проскальзываний (Slippage) для тестирования стратегии в real-time.
- [ ] **Live Trading (Bybit/Binance):** Подключение API ключей для отправки реальных ордеров.
- [ ] **Async Event Loop:** Бесконечный цикл, который по таймеру (Polling) или через WebSockets запрашивает данные и передает их в движок.

## 4. Интерфейс (TUI) и Аналитика
- [ ] **Backtester Dashboard:** Вывод результатов исторического тестирования (Winrate, Profit Factor, Max Drawdown).
- [ ] **Интеграция с Textual / Rich:** Построение терминального UI для отображения живых логов, открытых позиций и текущего баланса без необходимости смотреть в консольный вывод.
- [ ] **Логирование:** Переход от обычных `print()` к структурному логированию (модуль `logging`), запись логов в файлы с ротацией.
